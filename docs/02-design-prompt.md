请为一款桌面应用设计高保真可交互原型（用 Web 模拟桌面窗口，画布 1440×900，多屏可点击流转）。

## 产品背景

产品名 **CodeSucker · 软著代码抽取器**。一句话：把任意本地代码项目一键整理成符合中国软件著作权申报规范的 60 页源程序文档（Word），全程离线，代码不出本机。

目标用户是中小软件公司的研发或行政人员、独立开发者。他们一年只用几次、完全不想读申报规范、最怕材料被退回补正。产品的核心价值是**不懂规范也能一次通过**：规范已内置（前后各连续 30 页、每页 50 行、页眉=软件全称+版本号、页码 1–60、第 1 页必须是模块开头、第 60 页必须是模块结尾、无空行无注释），并在导出前自动校验。

## 设计风格

- 现代开发者工具气质，克制、精致，参考 Linear / VS Code / Raycast
- 全中文界面；浅色为默认主题，需另外展示一屏暗色主题效果
- 主色用靛蓝或青绿系；校验状态色：通过=绿、警告=橙、退回风险=红
- 代码区一律等宽字体（JetBrains Mono 风格）；A4 预览页内代码用宋体模拟 Word 效果
- 窗口有自绘标题栏：左侧 logo + 产品名，右侧最小化/最大化/关闭

## 信息架构

左侧固定竖向步骤导航（5 步向导，显示完成态勾号），顶部工具栏含：当前项目名下拉（可切换最近项目）、"保存配置"按钮、主题切换、设置图标（带更新红点）。

步骤：**1 导入项目 → 2 文件与排序 → 3 清洗与排版 → 4 分页预览 → 5 校验与导出**

## 逐屏要求

### 屏 1 · 导入项目

- 居中大号虚线拖拽区："拖入项目文件夹，或点击选择"，下方小字"完全离线处理，代码不会离开这台电脑 🔒"
- 下半部分"最近项目"卡片列表（示例数据）：
  - 智慧园区巡检管理系统 · ~/work/park-inspect · 上次生成 2026-07-18 · 60 页 ✅
  - 冷链物流温控平台 · ~/work/coldchain · 上次生成 2026-06-02 · 58 页 ⚠️
- 拖入后出现扫描进度态：进度条 + "正在扫描 1,284 个文件…已识别 Java 46 / Kotlin 12 / XML 30"

### 屏 2 · 文件与排序

三栏布局：

- 左栏：文件树，每个文件带勾选框、语言图标、行数徽标（如 `MainActivity.kt · 412 行`）；被默认排除的目录（node_modules、build、.git）灰显并标"已排除"
- 中栏：**已纳入文件的有序列表**，可拖拽调整顺序；顶部排序策略分段控件：`入口优先（推荐） / 修改时间 / 手动`；第一项带"📌 首页起点"标签（示例 `App.kt`），最后一项带"🏁 末页终点"标签
- 右栏统计面板：总文件 128 / 已纳入 46 / 清洗后总行数 8,642 / 预计页数 60 页（进度环）/ HTML+CSS 占比 12%（横向进度条，旁注"建议 ≤20%"）

### 屏 3 · 清洗与排版

左右分栏：

- 左侧配置表单：
  - "软件全称 + 版本号"输入框（必填红星，placeholder"须与申请表完全一致"，示例值：智慧园区巡检管理系统V1.0）
  - 开关组：删除注释（默认开）、删除空行（默认开）、敏感信息脱敏（默认开，副文案"API 密钥 / 密码 / 内网 IP / 手机号"）、超长行自动折行（默认开）
  - 排版参数（折叠面板，默认收起）：字体 宋体、字号 10.5、行距 固定值 10.5pt、每页行数 50
- 右侧实时预览：同一段代码"清洗前 → 清洗后"上下对比，被删除的注释行红色删除线，被脱敏的 `apiKey = "sk-****"` 高亮标注；顶部小字"预览文件：PatrolService.kt"

### 屏 4 · 分页预览

- 中央 A4 纸张仿真（带阴影浮起）：页眉左侧"智慧园区巡检管理系统V1.0"、右上角页码"3"，正文恰好 50 行宋体代码
- 纸张左右两侧有翻页箭头；底部横向缩略页条 1–60 可点击跳转，其中第 30/31 页之间画一个明显的"✂️ 前后段分界"标记，第 1 页标"模块开头 ✓"、第 60 页标"模块结尾 ✓"
- 右上角信息条：共 60 页 · 3,000 行 · 前段止于 PatrolService.kt · 后段起于 ReportExporter.kt

### 屏 5 · 校验与导出

- 主区域为校验清单卡片流，每条含状态图标、规则名、详情、可展开定位，示例必须包含这四种：
  - ✅ 每页行数均 ≥ 50 行
  - ✅ 页眉与软件名称一致
  - ⚠️ 末页仅 40 行，不足页面 2/3（建议补充或调整截取点）
  - ❌ PatrolService.kt 第 12 行检测到 `@author 张三`，与著作权人"某某科技有限公司"不一致（展开后显示代码上下文 + "一键移除"按钮）
- 顶部汇总横幅："2 项通过 · 1 项警告 · 1 项退回风险"，风险未清零时主导出按钮旁有提示
- 右侧导出面板：格式复选（docx 默认勾选、txt）、输出路径、大号主按钮"生成申报文档"
- 导出成功态：弹窗 ✅"已生成 源程序_智慧园区巡检管理系统V1.0.docx（60 页）"，按钮"打开所在文件夹 / 再次生成"

### 附加屏

- 设置页：更新检查（"当前 v1.2.0，检测到新版本 v1.3.0"+ 更新日志摘要 + "下载更新"按钮）、默认排除规则编辑、隐私说明（强调零网络请求）
