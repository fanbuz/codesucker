# CodeSucker 功能设计与技术选型

> v0.1.0 当前实现基线。CodeSucker 用本地桌面流程整理软件著作权源程序材料；它帮助减少格式与内容准备错误，但不替代登记机构审查，也不构成法律建议。

## 一、产品定位

- **一句话**：离线扫描本地工程，完成代码清洗、前后段截取、显式分页、风险校验与 docx/txt 导出。
- **目标用户**：需要准备软件著作权源程序鉴别材料的研发、行政人员、独立开发者和代理机构。
- **当前差异点**：五步图形向导、带跨行字符串状态的注释清洗、显式分页、导出前风险校验、源码处理全程离线。

## 二、v0.1.0 功能边界

### 核心流水线

```text
发现 Discover → 清洗 Clean → 排序截取 Select → 渲染 Render → 校验 Audit
```

| 模块 | 当前实现 |
|---|---|
| 项目导入 | 选择或拖入一个本地文件夹；支持最近项目入口 |
| 文件发现 | 后缀白名单；排除 `.git`、`node_modules`、`dist`、`build`、`vendor` 等目录；尊重 `.gitignore`；探测 UTF-8、GBK 等编码 |
| 文件管理 | 文件树勾选、入口优先/修改时间/手动排序、原始行数和 HTML+CSS 占比预估 |
| 清洗引擎 | 逐字符状态机处理单行/块注释、Python docstring 与跨行字符串；删除空行；Tab 转 4 空格；78 列硬折行；敏感信息脱敏 |
| 截取分页 | 总行数不超过 3000 时全量输出；超过时按有序代码流取前 1500 行和后 1500 行；首页从首个入选文件开头开始，末页以最后一个入选文件结尾；每 50 行显式分页 |
| 文档渲染 | docx 页眉、PAGE 页码域、宋体 10.5pt、固定行距与显式分页符；同步支持 txt 备查文件 |
| 风险校验 | 有效内容、页眉、每页行数、末页 2/3、首末页边界、残留空行、HTML+CSS 占比和署名冲突；根据输入条件展示对应项目 |
| 工程配置 | 将标题、著作权人、排序、文件选择、清洗开关和导出设置保存到 `.codesucker.json` |
| 桌面分发 | macOS ARM64 DMG、macOS x64 DMG、Windows x64 NSIS；首版未签名、公证 |

### 当前明确不包含

- tree-sitter 或完整语言语法树解析
- 多目录合并导入
- 著作权人成立日期的 UI 输入
- Linux 安装包
- 自动更新
- macOS/Windows 正式签名与公证
- 校验项一键修复、PDF、CLI 和例外交存模式

这些能力属于后续版本规划，不应在 README 或 UI 中表述为 v0.1.0 已实现功能。

## 三、技术实现

### 仓库结构

```text
packages/
  core/  纯 TypeScript：discover / clean / select / render / audit
  app/   Electron 43 + React 18 + Zustand + electron-vite
```

### 关键依赖

- 文件扫描：`fast-glob`、`ignore`
- 编码识别：`chardet`、`iconv-lite`
- 注释清洗：项目内逐字符状态机，不依赖 tree-sitter
- 文档生成：`docx`
- 桌面 UI：Electron、React、Zustand、自定义 CSS
- 打包分发：`electron-builder`，目标为 macOS DMG 与 Windows NSIS

### 关键决策

1. **显式分页**：先在内存中按 50 行组成页面，再写入分页符，不依赖字体和行距凑页。
2. **状态机清洗**：识别注释、普通字符串、模板字符串和 Python 三引号字符串边界，避免正则误删字符串内容。
3. **按行精确截取**：前后段各取固定行数；整个输出的首页与末页分别锚定首个和最后一个入选文件边界，不宣称中间截点按文件原子对齐。
4. **有限并发**：异步 I/O 配合 1～4 个 worker，保留稳定排序、取消语义和主线程响应。
5. **业务与壳分离**：核心流水线不依赖 Electron，便于测试和后续复用。
6. **默认离线**：扫描、读取、清洗、校验和导出不发起网络请求；用户主动打开外部链接不属于源码处理流程。

## 四、测试与发布基线

- `npm run verify`：版本、公共 lockfile、单元/回归测试、构建和 worker 集成
- `npm run test:real-projects -w @codesucker/core -- <java-kotlin> <python> <typescript>`：三类真实项目端到端验收
- `npm run perf`：5000 文件性能基线
- PR 打包门禁：macOS x64、macOS ARM64、Windows x64
- `v*` tag：自动生成安装包、`SHA256SUMS.txt` 与 GitHub Release

## 五、后续路线

- **V2**：Linux 安装包、签名与公证、自动更新、多目录导入、成立日期输入、自定义规则、一键修复、CLI
- **V3**：用户手册/设计说明书模板、多申报主体、例外交存模式
