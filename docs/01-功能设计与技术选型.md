# CodeSucker 功能设计与技术选型

> 软著代码抽取器：把任意本地代码项目一键整理成符合中国软件著作权申报规范的源程序文档（docx），全程离线，代码不出本机。

## 一、产品定位

- **一句话**：不懂申报规范，也能一次通过——规则内置、自动校验、离线生成。
- **目标用户**：中小软件公司研发/行政、独立开发者、代理机构。特点是一年用几次、不想读规范、最怕被退回补正。
- **差异化**（对比 CodeGenerator / pyerz 等现有工具）：
  1. Token 级注释剥离（不误伤字符串里的 `//`），全语言覆盖；
  2. 显式分页（每页恰好 50 行，不靠排版参数"凑"）；
  3. **提交前自动合规校验**——现有工具全部缺失的一环；
  4. 全平台 + 自动更新 + 图形界面。

## 二、功能设计

### 核心流水线（五段）

```
发现 Discover → 清洗 Clean → 排序截取 Select → 渲染 Render → 校验 Audit
```

### V1（MVP）

| 模块 | 功能点 |
|---|---|
| 项目导入 | 文件夹选择/拖拽、多目录、最近项目 |
| 文件发现 | 后缀白名单；默认黑名单（.git/node_modules/dist/build/vendor/*.min.*/测试目录）；尊重 .gitignore；编码探测（GBK/BOM → UTF-8） |
| 文件管理 | 文件树勾选纳入/排除、行数统计、排序策略（入口优先/修改时间/手动拖拽） |
| 清洗引擎 | tree-sitter token 级去注释（单行/块注释/docstring，字符串内容不误伤）；去空行；tab→4空格；超长行按 78 半角字符硬折行 |
| 截取分页 | 总行数 ≤3000 全量；>3000 取前后各 1500 行，**以文件为原子单位对齐边界**（首页=文件开头、末页=文件结尾）；每 50 行一页 |
| 文档渲染 | docx：页眉（软件名+版本号）、右上角 PAGE 页码域、宋体 10.5pt、固定行距、每页显式分页符；同步输出 txt 备查 |
| 合规校验 | 报告：每页≥50行 / 末页≥2/3 / 页眉一致 / 首末页模块边界 / 残留空行注释 / HTML+CSS 占比 / @author、Copyright 署名扫描 / 文件时间早于成立日期检测；三级结论（通过/警告/退回风险） |
| 工程配置 | `.codesucker.json` 持久化项目配置，重复生成结果一致 |

### V2

- 敏感信息扫描与脱敏（API key、密码、内网 IP、手机号 → 占位符替换 + 报告）
- 例外交存模式（黑斜线覆盖 ≤50%）
- PDF 导出；CLI 版本（复用 core 包）
- 自动更新（差量）

### V3

- 用户手册/设计说明书模板化生成、申请表字段辅助填写
- 多申报主体模板管理

## 三、技术选型

### 壳层对比

| 方案 | 优势 | 劣势 | 结论 |
|---|---|---|---|
| **Electron** | Node 全能力（fs/native 模块直用）、electron-updater 差量更新最成熟、生态和 AI 辅助开发资料最多、开发速度最快 | 安装包 ~90–120MB | ✅ **首选** |
| Tauri 2 | 安装包 ~10–20MB、内存占用小、内置 updater | fs 走插件桥、Rust 门槛、生态资料少 | 备选（体积敏感时迁移，core 包可完全复用） |
| Flutter Desktop | 一致渲染 | docx/词法分析生态弱 | 排除 |
| PWA/纯 Web | 零安装 | File System Access API 兼容性差、"代码不出本机"心智难建立 | 排除 |

### 推荐栈（Electron 路线）

- **架构**：monorepo，业务全部沉到纯 TS 包，壳只做 IO 和窗口
  ```
  packages/
    core/     # 纯 TypeScript：discover/clean/select/render/audit（零 Electron 依赖）
    app/      # Electron + React + Tailwind（electron-vite 脚手架）
    cli/      # V2：commander 包一层 core
  ```
- **关键库**：
  - 文件扫描：`fast-glob` + `ignore`（.gitignore 语义）
  - 编码：`chardet` + `iconv-lite`
  - 注释剥离：`web-tree-sitter`（WASM，主进程加载，覆盖 Java/Kotlin/C/C++/C#/Python/JS/TS/Go/Rust/PHP/Ruby/Swift/ObjC/Dart/Lua/Shell/HTML/CSS/SQL/Vue 等 20+ 语言）；无 grammar 的语言回退到带字符串状态机的单行注释解析
  - docx：`docx`（npm，纯 JS，支持页眉/PAGE 域/分页符/固定行距）
  - UI：React + Tailwind + shadcn/ui，状态用 zustand
  - 打包：electron-builder（msi/dmg/AppImage + deb）
- **更新通道**：electron-updater；考虑国内网络，发布产物同步到 **对象存储（阿里 OSS/腾讯 COS）静态托管** 作为主更新源，GitHub Releases 作为镜像。
- **隐私承诺**：默认零网络请求（更新检查手动/可关），作为核心卖点写进产品页。

### 关键技术决策记录

1. **显式分页而非排版凑页**：内存中按 50 行切块 + 分页符，固定行距只作兜底；前提是渲染前消灭折行（超长行硬折断）。
2. **注释剥离用词法分析而非正则**：`"https://..."` 场景是正则流派的必错题。
3. **截取以文件为原子单位装箱**：自动满足"第 1 页是模块开头、第 60 页是模块结尾"的审查要求。
4. **core 包零壳依赖**：未来 CLI/Web/Tauri 迁移零成本。

## 四、迭代路线

- **M1**：core 五段流水线（先支持 5 种主流语言）+ 最小 UI 跑通导入→导出
- **M2**：全语言 grammar、校验报告 UI、配置持久化、三平台打包
- **M3**：自动更新上线、脱敏、CLI
