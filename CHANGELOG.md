# Changelog

本文件记录 CodeSucker 的用户可见变化，格式遵循 Keep a Changelog，版本号遵循 Semantic Versioning。

## [Unreleased]

## [0.4.0] - 2026-07-23

### Added

- 应用重启后安全恢复普通窗口的尺寸、位置和最大化状态，显示器或缩放环境变化时自动将窗口收回可用工作区
- 最近项目支持右键置顶/取消置顶、单项移除和管理模式批量移除，并标记不存在或无法访问的项目路径

### Changed

- 最近项目使用带 schema 的 JSON 结构、原子写入和旧数据迁移；置顶项目优先排序且不占用普通最近记录限额
- 设置页移除干扰内容的卡片线框装饰，对齐双栏底部，并将超量排除规则限制在列表内部滚动

### Fixed

- 修复最小窗口下设置页出现整页纵向滚动、双栏底部错位与操作区域可能被挤出可视范围的问题

## [0.3.2] - 2026-07-23

### Added

- 已打开项目支持手动重新扫描；重扫保留未保存的软件信息、文件选择、排序、清洗和导出配置，并按相对路径合并新增、变更与删除文件
- 扫描、处理和导出引入 `scanSessionId` 会话门禁，旧任务晚到结果无法覆盖当前项目状态

### Changed

- 分页预览改为信息栏、文档舞台和缩略页操作栏三段式布局，A4 纸张随可用空间等比例缩放，低于可读阈值时由舞台独立滚动
- 导入页固定拖入、扫描进度和错误恢复区域，最近项目列表独立滚动并阻止滚动传播
- 重扫成功后回到文件与排序步骤，后续步骤必须按顺序重新确认和生成

### Fixed

- 修复源码在应用外部变化后仍可能显示或导出旧处理、分页、校验结果的问题
- 修复小窗口和系统缩放下分页信息、A4 文档、翻页按钮、缩略页条互相遮挡或挤压的问题
- 修复最近项目较多时导入、取消扫描和错误恢复操作被整体页面滚动卷走的问题

## [0.3.1] - 2026-07-23

### Added

- 校验结果中的问题文件与证据位置支持点击，并在系统文件管理器中安全定位到项目内文件

### Changed

- 设置页压缩顶部间距并固定交互标题区，规则列表与页面内容使用受控滚动；隐私说明和关于信息纵向排列
- 软件更新说明改为弹窗展示，支持键盘焦点约束、Esc 关闭和长内容独立滚动
- README 将 Apple Silicon 统一描述为 M 系列芯片，不再枚举具体型号

### Fixed

- 修复「清洗与排版」展开排版参数后内容被底部按钮遮挡、无法滚动到底部的问题
- 修复校验详情中的长文件路径和证据文本溢出卡片容器的问题

## [0.3.0] - 2026-07-23

### Added

- 「文件与排序」支持真实目录树、目录三态批量选择以及全局全选、清空和反选，并保留未纳入文件的手动排序位置
- 设置页支持新增、编辑、删除、去重和恢复应用级扫描排除规则；规则经安全校验后持久化，并从下一次项目扫描开始与 `.gitignore` 叠加生效

### Changed

- 合规规则版本提升至 `2026.07.2`，移除缺少正式依据的 HTML/CSS 占比校验；HTML/CSS 仍作为普通文件类型参与统计、清洗与导出

### Fixed

- 修复最小窗体和大量文件类型下右侧统计栏挤压、文字换行及“下一步”操作被推离可视区域的问题

## [0.2.0] - 2026-07-23

### Added

- 启动时自动查询 GitHub 最新正式 Release，并在设置页展示最新版、新版本或可恢复的失败状态
- 设置页支持手动重新检查更新，并可通过系统浏览器打开受信任的 Release 下载页面
- 「文件与排序」增加工程文件类型构成，支持全部/已纳入、代码行/文件数双口径统计
- 支持按后缀批量选择参与清洗与导出的源码，可一键仅保留 `.java` 等单一类型

### Changed

- 设置页改为居中的响应式布局，并重新绘制明暗主题切换图标
- 隐私说明明确区分离线源码处理与 GitHub 版本元数据查询
- HTML/CSS 风险占比改为从通用文件类型统计派生，并随文件选择即时更新
- README 增加未签名 macOS 安装包的 `xattr` 下载隔离处理说明

### Fixed

- 修复“已纳入”视图中折叠文件类型仍按全项目口径汇总的问题
- 修复临时按后缀筛选后手动文件顺序被永久丢失的问题
- 修复更新检测对 SemVer 构建元数据、预发布标签和大小写排序的处理

## [0.1.0] - 2026-07-23

### Added

- 五段代码抽取流水线：发现、清洗、截取分页、渲染和合规校验
- Electron 五步向导、项目配置持久化与 docx/txt 导出
- 产品版本、配置 schema 与合规规则的独立版本追踪
- 统一版本设置、版本一致性检查和 CI 门禁
- 大仓库扫描、清洗与文档渲染的 worker 线程流水线
- 可取消任务、真实阶段进度与单文件错误汇总
- 署名证据保留与跨语言合规回归测试
- 设置页关于区域，展示作者、免费软件属性、许可证与 GitHub 入口
- README 增加基于 Mochi Issue Flow 推进开发的说明
- macOS x64、macOS arm64 与 Windows x64 的可重复安装包构建和 GitHub Release 工作流

### Changed

- 项目许可证由 GPL-3.0 切换为 Apache-2.0
- 设置页版本号由构建时应用版本注入，不再硬编码
- `.codesucker.json` 保存时写入应用版本、配置 schema 和规则版本
- 合规规则版本提升至 `2026.07.1`，署名冲突改为依据清洗前保留的文件、行号、原文与主体信息判断
- 大仓库处理限制在 1–4 个 worker，并保证并发前后的文件顺序、截取结果与审计结果一致
- 运行与打包环境统一为 Node.js 22.12 或更高版本

### Fixed

- 修复删除注释后署名信息丢失，导致冲突申报漏报的问题
- 修复扫描、清洗和 DOCX 渲染长时间阻塞 Electron 主进程的问题
- 修复跨行模板字符串和 Python 三引号字符串中的注释符号被误删
- 修复 0 行、0 页结果被错误标记为可导出
- 修复 Electron 43 下拖入文件夹无法可靠取得路径

### Security

- 固定使用公共 npm 源，并阻止 lockfile 提交非公共依赖下载地址
