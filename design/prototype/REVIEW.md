# CodeSucker 原型复查记录

本目录保存 2026-07-23 复查后的界面原型。`CodeSucker.dc.html` 与 `uploads/` 是当前 UI 实现参考，旧版差异可通过 Git 历史查看。

## 相较上一版的变化

- 顶部主题切换改用月亮/太阳 SVG，设置入口改为更明确的齿轮图标。
- 「文件与排序」右侧不再只显示固定的 HTML + CSS 占比，改为文件类型构成、总行数和分段比例视图。
- 文件类型条目增加选择状态，点击后可以控制哪些后缀参与后续清洗与导出；统计会随选择即时更新。
- 文件类型构成随后缀选择即时更新；HTML、CSS 与其他源码类型采用相同的统计和选择逻辑。
- 文件树采用真实目录层级与目录三态选择，顶部集中提供全选、清空和反选。
- 右侧统计摘要和底部操作区保持固定，文件类型条目在可用高度内独立滚动。
- 设置页改为居中的双栏布局，增加应用身份与版本更新 Hero、结构化更新日志。
- 默认排除规则升级为可编辑列表，支持校验、去重、保存和恢复默认，并明确“下次扫描生效”及 `.gitignore`/项目文件选择的职责边界；隐私说明和关于信息保持独立分组。
- 原型包新增 5 张交互复查截图，覆盖文件类型全选、部分取消、统计变化和设置页新布局。

## 实现取舍

- 实际产品以“后缀”为最小选择单位，例如可以只保留 `.java`；语言名称作为辅助说明，避免同一语言的不同后缀被强制绑定。
- 统计同时保留“工程全部源码”和“当前已纳入源码”两个口径，并支持按文件数、代码行数查看。
- 类型较多时采用 Top N + 展开其余项，避免右栏随大仓库无限增长。
- 选择后缀只更新扫描结果中的 `included` 状态，不重新遍历目录或读取文件。
- 目录与全局选择同样只更新 `included`，完整手动顺序继续保留未纳入文件的位置。
